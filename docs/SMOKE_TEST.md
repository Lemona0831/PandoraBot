# PANDORA 개장 전 스모크 테스트

이 문서는 개장 전에 **핵심 기능이 정말 도는지** 빠르게 확인하려고 만든 체크리스트입니다.  
개발자 문서라기보다, 운영자가 마지막 점검할 때 보는 문서라고 생각하면 됩니다.

겁먹고 전부 꼼꼼하게 읽기보다, 위에서 아래로 하나씩 눌러본다는 느낌으로 보면 됩니다.

## 테스트 전에

- Discord 서버에 봇이 들어와 있어야 합니다.
- 테스트 계정에 운영 권한이 있어야 합니다.
- 테스트용 캐릭터 시트 1개 정도는 준비되어 있으면 편합니다.
- 테스트용 에너미 1개, 드롭 1개 정도가 있으면 좋습니다.

## 빠른 기준

- 정상: 명령을 쳤을 때 응답이나 Embed가 바로 뜬다.
- 확인 필요: 반응이 없거나, 권한/설정 안내가 뜬다.
- 문제: raw 에러 메시지가 뜨거나, 기록이 남아야 할 곳에 기록이 안 남는다.

## 1. 봇 실행 확인

- 목적: 봇이 살아 있는지 본다.
- 실행: `dotnet run` 또는 운영 중인 실행본 확인
- 기대 결과: Discord에서 봇이 온라인으로 보인다.
- 실패 시 확인:
  - `BotSettings.json`
  - Discord Token
  - Google 인증 파일 경로

## 2. Slash Command 등록 확인

- 목적: 명령어가 Discord에 보이는지 본다.
- 실행: Discord 입력창에 `/`
- 기대 결과: PANDORA 명령들이 뜬다.
- 실패 시 확인:
  - 봇 재실행
  - Guild ID
  - `applications.commands` 권한

## 3. `/등록`

- 목적: 플레이어 캐릭터 등록이 되는지 확인
- 실행: `/등록 시트:<테스트시트URL>`
- 기대 결과: 등록 완료 안내가 뜬다.
- 실패 시 확인:
  - 시트 URL
  - 서비스 계정 접근 권한
  - 원본 시트 구조

## 4. `/검수상태`

- 목적: 등록한 캐릭터의 검수 상태가 보이는지 확인
- 실행: `/검수상태`
- 기대 결과: `pending` 또는 해당 상태가 뜬다.
- 실패 시 확인:
  - `/등록`이 실제로 됐는지
  - 같은 계정으로 테스트 중인지

## 5. `/관리검수목록`

- 목적: 운영자가 검수 대기 목록을 볼 수 있는지 확인
- 실행: `/관리검수목록 상태:pending`
- 기대 결과: 대기 캐릭터 목록이 뜬다.
- 실패 시 확인:
  - 운영 권한
  - 등록된 캐릭터 상태

## 6. `/관리승인`

- 목적: 승인 처리가 되는지 확인
- 실행: `/관리승인 캐릭터:<캐릭터이름>`
- 기대 결과: 승인 완료 안내가 뜬다.
- 실패 시 확인:
  - 이름이 정확한지
  - 같은 이름이 여러 개인지

## 7. `/선택`

- 목적: 플레이어가 캐릭터를 선택할 수 있는지 확인
- 실행: `/선택 캐릭터:<캐릭터이름>`
- 기대 결과: 선택 완료 안내가 뜬다.
- 실패 시 확인:
  - 승인 상태인지
  - 본인 캐릭터인지

## 8. `/현재`

- 목적: 현재 선택 캐릭터 정보가 뜨는지 확인
- 실행: `/현재`
- 기대 결과: HP와 능력치가 뜬다.
- 실패 시 확인:
  - `/선택`을 했는지

## 9. `/정보`

- 목적: 상태창 이미지가 뜨는지 확인
- 실행: `/정보`
- 기대 결과: 상태창 이미지가 올라온다.
- 실패 시 확인:
  - 파일 업로드 권한
  - 캐릭터 데이터가 비어 있지 않은지

## 10. `/판정`

- 목적: 판정과 기록이 정상인지 확인
- 실행: `/판정 능력치:근력`
- 기대 결과: 결과 Embed가 뜬다.
- 실패 시 확인:
  - 선택된 캐릭터가 있는지
  - 능력치 이름을 맞게 넣었는지

## 11. `/피해`

- 목적: 운영자가 캐릭터 HP를 줄일 수 있는지 확인
- 실행: `/피해 캐릭터:<이름> 피해:1`
- 기대 결과: HP가 줄고 안내가 뜬다.
- 실패 시 확인:
  - 운영 권한
  - 대상 이름

## 12. `/회복`

- 목적: 운영자가 캐릭터 HP를 회복시킬 수 있는지 확인
- 실행: `/회복 캐릭터:<이름> 회복:1`
- 기대 결과: HP가 회복된다.
- 실패 시 확인:
  - 최대 HP 계산이 이상하지 않은지

## 13. `/내정보`

- 목적: 플레이어 정보 모아보기가 되는지 확인
- 실행: `/내정보`
- 기대 결과: 현재 캐릭터, 보유 캐릭터, 최근 기록이 뜬다.
- 실패 시 확인:
  - 등록/선택 상태

## 14. `/관리목록`

- 목적: 운영자가 전체 캐릭터 목록을 볼 수 있는지 확인
- 실행: `/관리목록`
- 기대 결과: 캐릭터 목록 Embed가 뜬다.
- 실패 시 확인:
  - 운영 권한

## 15. `/정보조회`

- 목적: 운영자가 특정 캐릭터 상세 정보를 볼 수 있는지 확인
- 실행: `/정보조회 캐릭터:<이름>`
- 기대 결과: 캐릭터 상세 정보가 뜬다.
- 실패 시 확인:
  - 이름 입력
  - 중복 이름 여부

## 16. `/공지`

- 목적: 공지 발송과 기록이 되는지 확인
- 실행: `/공지 종류:테스트 제목:스모크 테스트 내용:공지 확인`
- 기대 결과: 공지가 올라가고 기록이 남는다.
- 실패 시 확인:
  - 채널 권한
  - 멘션 권한

## 17. `/에너미목록`

- 목적: 등록된 에너미 목록을 볼 수 있는지 확인
- 실행: `/에너미목록`
- 기대 결과: 에너미 목록이 뜬다.
- 실패 시 확인:
  - 에너미 데이터가 있는지

## 18. `/에너미조회`

- 목적: 특정 에너미 상세 조회가 되는지 확인
- 실행: `/에너미조회 에너미:<이름또는ID>`
- 기대 결과: 상세 정보가 뜬다.
- 실패 시 확인:
  - 이름 검색어를 너무 넓게 넣지 않았는지

## 19. `/에너미판정`

- 목적: 에너미 판정이 되는지 확인
- 실행: `/에너미판정 에너미:<이름또는ID> 능력치:근력`
- 기대 결과: 결과 Embed가 뜬다.
- 실패 시 확인:
  - 능력치 이름
  - 검색 결과가 한 개로 좁혀지는지

## 20. `/드롭`

- 목적: 실제 드롭 결과가 나오는지 확인
- 실행: `/드롭 에너미:<이름또는ID> 수량:1`
- 기대 결과: 드롭 결과 또는 전리품 없음이 뜬다.
- 실패 시 확인:
  - 드롭 테이블
  - 드롭 설정

## 21. `/드롭테스트`

- 목적: 드롭 확률 테스트가 되는지 확인
- 실행: `/드롭테스트 에너미:<이름또는ID> 반복횟수:100`
- 기대 결과: 아이템별 횟수와 비율이 뜬다.
- 실패 시 확인:
  - 반복횟수 범위
  - 드롭 설정 존재 여부

## 22. `/전투보조`

- 목적: 운영 흐름 안내가 잘 뜨는지 확인
- 실행: `/전투보조`
- 기대 결과: 전투 진행 순서 Embed가 뜬다.
- 실패 시 확인:
  - 운영 권한

## 23. `/로그`

- 목적: 운영 기록이 실제로 남는지 확인
- 실행:
  - `/로그 로그대상:전체`
  - `/로그 로그대상:전투`
  - `/로그 로그대상:관리`
- 기대 결과: 최근 기록이 보인다.
- 실패 시 확인:
  - DB 연결
  - 해당 명령을 실제로 실행했는지

## 전투 쪽 간단 체크

전투 기능은 아래 순서로 한 번만 돌려보면 감이 옵니다.

1. `/전투시작 제목:스모크 테스트`
2. `/전투참가 캐릭터:<이름>`
3. `/에너미소환 에너미:<이름> 수량:1`
4. `/전투상태`
5. `/전투피해`
6. `/전투회복`
7. `/전투정리`
8. `/전투종료`

여기까지 자연스럽게 되면 개장 전 최소 전투 흐름은 통과로 봐도 됩니다.
