# PANDORA 봇 개장 전 스모크 테스트

이 문서는 진행자가 서버 개장 전에 PANDORA 봇의 핵심 기능이 정상 동작하는지 순서대로 확인하기 위한 점검표입니다.

실제 Discord Token, Google 서비스 계정 키, private key 값은 이 문서에 적지 않습니다.

## 테스트 전 준비

- Discord 봇이 테스트 서버에 초대되어 있어야 합니다.
- 진행자 계정에 관리자/진행자 권한이 있어야 합니다.
- Google Sheets 저장소 접근 권한이 설정되어 있어야 합니다.
- 테스트용 캐릭터 시트 URL 1개와 테스트용 에너미 1개가 준비되어 있어야 합니다.
- 테스트용 공지 채널에서 공지 발송을 확인할 수 있어야 합니다.

## 빠른 판정 기준

- 정상: 명령 실행 후 Discord 응답 또는 Embed가 표시됩니다.
- 확인 필요: 응답이 없거나, `Error:` 메시지가 나오거나, Google Sheets에 기록이 남지 않습니다.
- 치명: 봇이 오프라인이거나 Slash Command가 보이지 않습니다.

## 1. 봇 실행 확인

- 목적: 봇 프로세스가 정상 실행 중인지 확인합니다.
- 실행 명령: 로컬에서 `dotnet run` 또는 배포 환경의 봇 실행 절차를 수행합니다.
- 기대 결과: 콘솔에 로그인/시작 로그가 표시되고 Discord 서버에서 봇이 온라인으로 보입니다.
- 실패 시 확인할 것:
  - `BotSettings.json`이 존재하는지 확인합니다.
  - Discord Token이 올바른지 확인합니다.
  - Google 서비스 계정 파일 경로가 올바른지 확인합니다.
  - 봇이 Discord 서버에 초대되어 있는지 확인합니다.

## 2. Slash Command 등록 확인

- 목적: Discord Slash Command가 서버에 등록되었는지 확인합니다.
- 실행 명령: Discord 입력창에 `/`를 입력하고 PANDORA 명령 목록을 확인합니다.
- 기대 결과: `/등록`, `/판정`, `/관리목록`, `/에너미목록`, `/드롭`, `/전투보조` 등이 보입니다.
- 실패 시 확인할 것:
  - 봇을 재시작합니다.
  - 설정의 Guild ID가 테스트 서버와 일치하는지 확인합니다.
  - 봇에게 `applications.commands` 권한이 있는지 확인합니다.

## 3. /등록

- 목적: 플레이어 캐릭터를 저장소에 등록할 수 있는지 확인합니다.
- 실행 명령: `/등록 시트:<테스트 캐릭터 시트 URL>`
- 기대 결과: 등록 완료 메시지가 표시되고 Google Sheets의 캐릭터 저장소에 행이 추가됩니다.
- 실패 시 확인할 것:
  - 캐릭터 시트 URL이 올바른지 확인합니다.
  - 서비스 계정이 원본 시트와 저장소 시트에 접근 가능한지 확인합니다.
  - 원본 시트의 이름/능력치/HP 위치가 기대 구조와 맞는지 확인합니다.

## 4. /검수상태

- 목적: 플레이어가 본인의 캐릭터 검수 상태를 확인할 수 있는지 확인합니다.
- 실행 명령: `/검수상태`
- 기대 결과: 방금 등록한 캐릭터가 `pending` 또는 해당 검수 상태로 표시됩니다.
- 실패 시 확인할 것:
  - `/등록`이 성공했는지 확인합니다.
  - Discord User ID가 저장소에 기록되어 있는지 확인합니다.

## 5. /관리검수목록

- 목적: 진행자가 검수 대기 캐릭터 목록을 볼 수 있는지 확인합니다.
- 실행 명령: `/관리검수목록 상태:pending`
- 기대 결과: 검수 대기 캐릭터 목록이 표시됩니다.
- 실패 시 확인할 것:
  - 실행 계정에 관리자/진행자 권한이 있는지 확인합니다.
  - 저장소의 검수 상태 값이 `pending`인지 확인합니다.

## 6. /관리승인

- 목적: 진행자가 캐릭터를 승인할 수 있는지 확인합니다.
- 실행 명령: `/관리승인 캐릭터:<캐릭터 이름>`
- 기대 결과: 해당 캐릭터의 검수 상태가 `approved`로 변경됩니다.
- 실패 시 확인할 것:
  - 캐릭터 이름이 정확한지 확인합니다.
  - 같은 이름의 캐릭터가 여러 개 있는지 확인합니다.
  - 관리 로그 시트 쓰기 권한을 확인합니다.

## 7. /선택

- 목적: 플레이어가 승인된 캐릭터를 현재 캐릭터로 선택할 수 있는지 확인합니다.
- 실행 명령: `/선택 캐릭터:<캐릭터 이름>`
- 기대 결과: 선택 완료 메시지가 표시되고 선택 상태 시트가 갱신됩니다.
- 실패 시 확인할 것:
  - 캐릭터가 `approved` 상태인지 확인합니다.
  - 명령 실행자가 해당 캐릭터의 소유자인지 확인합니다.

## 8. /현재

- 목적: 현재 선택 캐릭터의 기본 상태를 확인할 수 있는지 확인합니다.
- 실행 명령: `/현재`
- 기대 결과: 선택된 캐릭터의 HP와 능력치가 표시됩니다.
- 실패 시 확인할 것:
  - `/선택`을 먼저 실행했는지 확인합니다.
  - 선택 상태 시트에 현재 캐릭터가 기록되어 있는지 확인합니다.

## 9. /정보

- 목적: 캐릭터 상태창 이미지가 정상 생성되는지 확인합니다.
- 실행 명령: `/정보`
- 기대 결과: HUNTER LICENSE 상태창 이미지가 Discord에 표시됩니다.
- 실패 시 확인할 것:
  - 봇이 파일 업로드 권한을 가지고 있는지 확인합니다.
  - 이미지 리소스 파일이 출력 폴더에 복사되었는지 확인합니다.
  - 캐릭터 HP와 능력치 값이 비어 있지 않은지 확인합니다.

## 10. /판정

- 목적: 플레이어 캐릭터 판정과 판정 로그 기록을 확인합니다.
- 실행 명령: `/판정 능력치:근력`
- 기대 결과: 2d6 판정 결과 Embed가 표시되고 판정 로그 시트에 기록됩니다.
- 실패 시 확인할 것:
  - `/선택`이 완료되어 있는지 확인합니다.
  - 능력치 이름을 `근력`, `민첩`, `체력`, `지능`, `지혜`, `매력` 중 하나로 입력했는지 확인합니다.
  - 판정 로그 시트 쓰기 권한을 확인합니다.

## 11. /피해

- 목적: 진행자가 캐릭터 HP를 감소시킬 수 있는지 확인합니다.
- 실행 명령: `/피해 캐릭터:<캐릭터 이름> 수치:1`
- 기대 결과: HP가 1 감소하고 관리 로그에 기록됩니다.
- 실패 시 확인할 것:
  - 실행 계정에 관리자/진행자 권한이 있는지 확인합니다.
  - 캐릭터 이름이 중복되지 않았는지 확인합니다.
  - 관리 로그 시트 쓰기 권한을 확인합니다.

## 12. /회복

- 목적: 진행자가 캐릭터 HP를 회복시킬 수 있는지 확인합니다.
- 실행 명령: `/회복 캐릭터:<캐릭터 이름> 수치:1`
- 기대 결과: HP가 회복되고 최대 HP를 넘지 않습니다.
- 실패 시 확인할 것:
  - 캐릭터의 최대 HP 값이 올바른지 확인합니다.
  - 저장소의 현재 HP 값이 숫자인지 확인합니다.

## 13. /내정보

- 목적: 플레이어가 자신의 현재 상태와 최근 기록을 확인할 수 있는지 확인합니다.
- 실행 명령: `/내정보`
- 기대 결과: 현재 선택 캐릭터, 보유 캐릭터, 최근 판정 기록이 표시됩니다.
- 실패 시 확인할 것:
  - 해당 유저가 등록한 캐릭터가 있는지 확인합니다.
  - 선택 상태와 판정 로그 시트를 확인합니다.

## 14. /관리목록

- 목적: 진행자가 전체 캐릭터 목록을 확인할 수 있는지 확인합니다.
- 실행 명령: `/관리목록 개수:20`
- 기대 결과: 캐릭터 목록 Embed가 표시됩니다.
- 실패 시 확인할 것:
  - 관리자/진행자 권한을 확인합니다.
  - 캐릭터 저장소에 데이터가 있는지 확인합니다.

## 15. /관리정보

- 목적: 진행자가 특정 캐릭터 상세 정보를 확인할 수 있는지 확인합니다.
- 실행 명령: `/관리정보 캐릭터:<캐릭터 이름>`
- 기대 결과: 해당 캐릭터의 HP와 능력치 정보가 표시됩니다.
- 실패 시 확인할 것:
  - 캐릭터 이름을 정확히 입력했는지 확인합니다.
  - 같은 이름의 캐릭터가 여러 명인지 확인합니다.

## 16. /공지

- 목적: 진행자가 공지를 발송하고 기록할 수 있는지 확인합니다.
- 실행 명령: `/공지 종류:테스트 제목:스모크테스트 내용:공지 발송 테스트`
- 기대 결과: 공지 Embed가 채널에 표시되고 공지 로그에 기록됩니다.
- 실패 시 확인할 것:
  - 봇이 채널에 메시지를 보낼 권한이 있는지 확인합니다.
  - 역할/유저/전체 멘션 옵션을 사용했다면 멘션 권한을 확인합니다.
  - 공지 로그 시트 쓰기 권한을 확인합니다.

## 17. /에너미목록

- 목적: 진행자가 등록된 에너미 목록을 확인할 수 있는지 확인합니다.
- 실행 명령: `/에너미목록`
- 기대 결과: 에너미ID, 이름, HP, 출현구분이 Embed로 표시됩니다.
- 실패 시 확인할 것:
  - 에너미 저장소에 테스트 에너미가 있는지 확인합니다.
  - 출현구분 필터를 사용했다면 값이 실제 데이터와 맞는지 확인합니다.

## 18. /에너미조회

- 목적: 진행자가 특정 에너미 상세 정보를 확인할 수 있는지 확인합니다.
- 실행 명령: `/에너미조회 에너미:<에너미ID 또는 이름 일부>`
- 기대 결과: 에너미 상세 정보, 드롭 테이블 여부, 드롭 설정 여부가 표시됩니다.
- 실패 시 확인할 것:
  - 검색어가 너무 넓어 후보가 여러 명인지 확인합니다.
  - 에너미ID를 직접 입력해 다시 시도합니다.

## 19. /에너미판정

- 목적: 진행자가 에너미 능력치 기반 판정을 굴릴 수 있는지 확인합니다.
- 실행 명령: `/에너미판정 에너미:<에너미ID 또는 이름 일부> 능력치:근력`
- 기대 결과: 에너미 판정 결과 Embed가 표시되고 관리 로그에 기록됩니다.
- 실패 시 확인할 것:
  - 능력치 이름이 올바른지 확인합니다.
  - 에너미 검색 결과가 하나로 좁혀지는지 확인합니다.

## 20. /드롭

- 목적: 실전 보상용 드롭 결과를 굴릴 수 있는지 확인합니다.
- 실행 명령: `/드롭 에너미:<에너미ID 또는 이름 일부> 수량:1`
- 기대 결과: 드롭 결과 Embed가 표시되고 관리 로그에 기록됩니다.
- 실패 시 확인할 것:
  - 해당 에너미에 드롭 테이블이 연결되어 있는지 확인합니다.
  - 드롭 설정의 발생률과 드롭 횟수가 올바른지 확인합니다.
  - 전리품이 나오지 않는 것은 정상 결과일 수 있습니다.

## 21. /드롭테스트

- 목적: 드롭 확률을 반복 시뮬레이션으로 확인합니다.
- 실행 명령: `/드롭테스트 에너미:<에너미ID 또는 이름 일부> 반복횟수:100`
- 기대 결과: 실제 보상 지급이 아닌 테스트 결과가 아이템별 획득 수량과 비율로 표시됩니다.
- 실패 시 확인할 것:
  - 반복횟수가 1 이상인지 확인합니다.
  - 최대 1000회 제한이 적용되는지 확인합니다.
  - 드롭 테이블과 드롭 설정을 확인합니다.

## 22. /전투보조

- 목적: 진행자가 수동 전투 운영 순서를 Discord에서 바로 확인합니다.
- 실행 명령: `/전투보조`
- 기대 결과: 에너미 확인, 판정, HP 조작, 보상 굴림 순서가 Embed로 표시됩니다.
- 실패 시 확인할 것:
  - Slash Command 등록 상태를 확인합니다.
  - 실행 계정의 관리자/진행자 권한을 확인합니다.

## 23. Google Sheets 로그 기록 확인

- 목적: 운영 기록이 Google Sheets에 정상 누적되는지 확인합니다.
- 실행 명령: Google Sheets에서 다음 탭을 직접 확인합니다.
- 기대 결과:
  - `판정 로그`에 `/판정` 기록이 남습니다.
  - `관리 로그`에 `/피해`, `/회복`, `/에너미판정`, `/드롭`, `/드롭테스트` 기록이 남습니다.
  - `공지 로그`에 `/공지` 기록이 남습니다.
- 실패 시 확인할 것:
  - 서비스 계정이 대상 스프레드시트 편집 권한을 가지고 있는지 확인합니다.
  - 탭 이름이 코드에서 사용하는 이름과 일치하는지 확인합니다.
  - 봇 콘솔에 Google API 오류가 출력되었는지 확인합니다.

## 개장 전 최종 체크

- 핵심 플레이어 흐름: `/등록` -> `/관리승인` -> `/선택` -> `/현재` -> `/판정`
- 핵심 진행자 흐름: `/관리목록` -> `/피해` -> `/회복` -> `/공지`
