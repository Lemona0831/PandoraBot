# PANDORA 1.0 전투 보조 범위

PANDORA 1.0의 전투 관련 기능은 전투 세션 자동화가 아니라 **진행자가 수동으로 전투를 운영할 수 있도록 돕는 보조 기능**을 목표로 한다.

즉, 봇은 전투 흐름을 대신 관리하지 않는다. 진행자가 상황을 판단하고 선언하며, 봇은 필요한 정보 조회, 판정, HP 조작, 드롭 굴림을 빠르게 처리하는 도구로 사용한다.

## 1.0 목표

- 진행자가 Discord 안에서 에너미와 플레이어 상태를 빠르게 확인한다.
- 진행자가 필요한 순간에 플레이어 판정과 에너미 판정을 굴린다.
- 진행자가 수동으로 판단한 피해/회복을 봇 명령으로 반영한다.
- 전투 종료 후 에너미 드롭 테이블을 기준으로 보상 결과를 굴린다.
- 전투 참가자 확인은 전체 선택 상태가 아니라 **활성 전투 참가자**를 기준으로 한다.

## 포함 기능

### 에너미 목록 / 조회

- `/에너미목록`
- `/에너미조회`

진행자가 전투에 사용할 에너미의 ID, 이름, HP, 능력치, 출현구분, 드롭 설정 여부를 확인한다.

### 에너미 판정

- `/에너미판정`

에너미 능력치 기준으로 `2d6 + 능력치 수정치` 판정을 굴린다. 플레이어 `/판정`과 같은 수정치 공식을 사용한다.

### 드롭 굴림

- `/드롭`
- `/드롭테스트`

전투 종료 후 실전 보상용 드롭을 굴리거나, 개장 전/운영 중 밸런스 확인을 위해 드롭 확률을 시뮬레이션한다.

### 활성 전투 참가자 관리

1.0에서 전투 상태 확인은 서버 전체 선택 캐릭터를 대상으로 하지 않는다. 진행자가 현재 전투에 참여시키는 캐릭터만 **활성 전투 참가자**로 다루는 것을 목표로 한다.

이 범위는 “전투 세션 자동화”가 아니라, 진행자가 수동 전투 중 필요한 참가자만 추려 보기 위한 보조 기능이다.

### 전투 상태 확인

- `/전투상태`

`/전투상태`는 `/선택` 상태 전체를 그대로 보여주는 명령이 아니다.  
반드시 **활성 전투 참가자만** 대상으로 상태를 확인한다.

따라서 다음과 같은 동작은 1.0 범위에서 피한다.

- 서버의 모든 선택 캐릭터를 전투 참가자로 간주하지 않는다.
- 단순히 `/선택 상태` 시트 전체를 전투 상태로 보여주지 않는다.
- 현재 전투에 포함되지 않은 플레이어 캐릭터를 전투 상태에 섞지 않는다.

### 전투 피해 / 회복

- `/피해`
- `/회복`
- `/관리체력`

전투 중 발생한 피해와 회복은 진행자가 판단하고, 봇은 HP 값을 반영하고 관리 로그에 기록한다.

## 제외 기능

PANDORA 1.0에서는 다음 기능을 제공하지 않는다.

- `/전투시작`
- `/전투종료`
- 턴 / 라운드 자동 관리
- 상태이상 자동 처리
- 에너미 AI
- 전투 로그 요약

이 기능들은 전투를 “세션 시스템”으로 자동화하는 범위에 가깝기 때문에 1.0 개장 안정판에서는 제외한다.

## 운영 기준

1.0 전투 운영의 책임은 진행자에게 있다.

- 진행자는 어떤 캐릭터와 에너미가 전투에 참여하는지 직접 판단한다.
- 진행자는 어떤 판정이 필요한지 직접 결정한다.
- 진행자는 피해량, 회복량, 상태 변화, 결과 서술을 직접 결정한다.
- 봇은 조회, 판정, HP 조작, 드롭 굴림을 빠르게 처리한다.

## 후속 버전 검토

1.5 이후에는 웹 대시보드와 봇 명령을 함께 고려해 활성 전투 참가자 관리 UX를 정리한다.

2.0 이후에는 전투 세션, 턴/라운드, 전투 종료 처리, 로그 요약 등을 별도 기능으로 검토할 수 있다. 다만 이 기능들은 1.0 개장 안정판의 필수 범위가 아니다.
