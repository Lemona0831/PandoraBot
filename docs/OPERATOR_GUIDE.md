# PANDORA 운영자 명령 안내서

이 문서는 **비개발자 운영자**를 기준으로 작성했습니다.  
코드를 몰라도, Discord 안에서 어떤 명령을 어떤 순서로 쓰면 되는지 빠르게 확인할 수 있게 정리했습니다.

## 먼저 알아둘 점

- `/등록`, `/갱신`은 **플레이어가 자신의 원본 캐릭터 시트**를 기준으로 사용합니다.
- 그 뒤의 조회, 검수, HP 조작, 전투, 에너미, 드롭, 로그 확인은 **운영자 명령**으로 관리합니다.
- 전투는 **세션 단위**로 운영합니다.
- 다만 PANDORA 1.0은 턴제 전투 시스템이 아닙니다.
  - 턴
  - 라운드
  - 이니셔티브
  - 행동완료 체크
  
  는 자동으로 관리하지 않습니다.

## 가장 자주 쓰는 운영 흐름

### 1. 캐릭터 검수

1. 플레이어가 `/등록`
2. 운영자가 `/관리검수목록`
3. 운영자가 `/관리승인` 또는 `/관리반려`
4. 플레이어가 `/선택`

### 2. 세션 중 캐릭터 상태 확인

1. `/관리목록`
2. 필요하면 `/정보조회`
3. HP를 조정해야 하면 `/피해`, `/회복`, `/체력설정`

### 3. 전투 시작부터 종료까지

1. `/전투시작`
2. `/전투참가`
3. `/에너미소환`
4. `/전투상태`
5. `/판정`, `/에너미판정`
6. `/전투피해`, `/전투회복`
7. 필요하면 `/전투퇴장`, `/전투정리`
8. 보상은 `/드롭`
9. 테스트 확인은 `/드롭테스트`
10. 끝나면 `/전투종료`

## 플레이어 관련 명령

운영자가 직접 쓰기보다, 플레이어가 어떤 흐름으로 쓰는지 알아둘 때 보면 좋습니다.

### `/등록`

- 용도: 원본 Google Sheets 캐릭터 시트를 읽어 캐릭터를 등록합니다.
- 누가 쓰나: 플레이어
- 예시:  
  `/등록 시트:구글시트주소`
- 결과:
  - 같은 시트를 다시 등록하면 새 캐릭터가 생기지 않고 기존 캐릭터가 갱신됩니다.
  - 등록 뒤에는 검수가 필요합니다.

### `/갱신`

- 용도: 같은 원본 시트를 다시 읽어 캐릭터 정보를 갱신합니다.
- 누가 쓰나: 플레이어
- 예시:  
  `/갱신`
- 결과:
  - 원본 시트 내용이 바뀌었을 때 최신 상태를 다시 반영합니다.

### `/검수상태`

- 용도: 내가 등록한 캐릭터가 승인됐는지 확인합니다.
- 누가 쓰나: 플레이어

### `/선택`

- 용도: 승인된 캐릭터 중 현재 사용할 캐릭터를 고릅니다.
- 누가 쓰나: 플레이어
- 예시:  
  `/선택 캐릭터:레모나`

### `/현재`

- 용도: 현재 선택한 캐릭터의 HP와 능력치를 간단히 봅니다.

### `/정보`

- 용도: 현재 선택한 캐릭터의 상태창 이미지를 봅니다.

### `/판정`

- 용도: 현재 선택한 캐릭터로 2d6 판정을 굴립니다.
- 채널에 보여도 되는 명령입니다.
- 예시:  
  `/판정 능력치:근력`

### `/내정보`

- 용도: 현재 캐릭터, 보유 캐릭터, 최근 기록을 한 번에 확인합니다.

## 캐릭터 운영 명령

### `/관리검수목록`

- 용도: 대기, 승인, 반려 상태의 캐릭터를 확인합니다.
- 언제 쓰나: 플레이어 등록 후 검수할 때
- 예시:  
  `/관리검수목록 상태:pending`

### `/관리승인`

- 용도: 특정 캐릭터를 승인 상태로 바꿉니다.
- 예시:  
  `/관리승인 캐릭터:레모나`

### `/관리반려`

- 용도: 특정 캐릭터를 반려 상태로 바꿉니다.
- 예시:  
  `/관리반려 캐릭터:레모나`

### `/관리목록`

- 용도: 등록된 캐릭터들을 한 번에 봅니다.
- 언제 쓰나: 세션 전 준비, 운영 중 인원 확인

### `/정보조회`

- 용도: 특정 캐릭터의 정보를 자세히 봅니다.
- 예시:  
  `/정보조회 캐릭터:레모나`

### `/체력설정`

- 용도: 현재 HP를 숫자로 바로 지정합니다.
- 예시:  
  `/체력설정 캐릭터:레모나 체력:9`

### `/피해`

- 용도: 특정 캐릭터에게 피해를 적용합니다.
- 예시:  
  `/피해 캐릭터:레모나 피해:2`

### `/회복`

- 용도: 특정 캐릭터를 회복합니다.
- 예시:  
  `/회복 캐릭터:레모나 회복:1`

### `/관리선택해제`

- 용도: 특정 캐릭터의 선택 상태를 해제합니다.
- 언제 쓰나: 잘못 선택됐거나 운영상 초기화가 필요할 때

### `/관리삭제`

- 용도: 특정 캐릭터를 삭제합니다.
- 주의: 실제 운영 데이터에 영향을 주므로 꼭 확인 후 사용하세요.

## 공지와 로그

### `/공지`

- 용도: 운영 공지를 발송하고 기록합니다.
- 언제 쓰나: 세션 시작 안내, 모집 공지, 중요 공지 전달

### `/로그`

- 용도: 판정, 관리, 공지, 전투 로그를 한곳에서 확인합니다.
- 예시:
  - `/로그 로그대상:전체`
  - `/로그 로그대상:전투`
  - `/로그 로그대상:공지`
- 언제 쓰나:
  - 누가 어떤 판정을 했는지 확인할 때
  - HP 조작 기록을 다시 볼 때
  - 전투 진행 기록을 되짚을 때

## 에너미 운영 명령

### `/에너미목록`

- 용도: 등록된 에너미를 목록으로 봅니다.
- 예시:
  - `/에너미목록`
  - `/에너미목록 출현구분:1층`

### `/에너미조회`

- 용도: 특정 에너미의 상세 정보를 봅니다.
- 예시:  
  `/에너미조회 에너미:슬라임`

### `/에너미판정`

- 용도: 에너미 능력치 기준으로 2d6 판정을 굴립니다.
- 예시:  
  `/에너미판정 에너미:슬라임 능력치:근력`

### `/에너미추가`

- 용도: 새 에너미를 등록합니다.
- 예시:  
  `/에너미추가 이름:슬라임 최대체력:8 출현구분:1층`

### `/에너미수정`

- 용도: 에너미의 기본 정보를 수정합니다.

### `/에너미능력치`

- 용도: 에너미 능력치 하나를 바로 조정합니다.

### `/에너미비활성화`

- 용도: 에너미를 목록과 운영에서 숨깁니다.
- 장점:
  - 예전 기록은 남기고
  - 나중에 다시 살릴 수 있습니다.

### `/에너미활성화`

- 용도: 비활성화된 에너미를 다시 사용 가능하게 바꿉니다.

## 드롭 운영 명령

### `/드롭`

- 용도: 실제 전투 종료 후 전리품을 굴립니다.
- 예시:  
  `/드롭 에너미:슬라임 수량:3`

### `/드롭테스트`

- 용도: 실제 지급이 아니라 확률 테스트를 돌립니다.
- 예시:  
  `/드롭테스트 에너미:슬라임 반복횟수:100`

### `/드롭추가`

- 용도: 드롭 아이템을 등록합니다.

### `/드롭목록`

- 용도: 에너미에 연결된 드롭 아이템을 확인합니다.
- 예시:  
  `/드롭목록 에너미:슬라임`

### `/드롭수정`

- 용도: 드롭 아이템 정보를 수정합니다.

### `/드롭삭제`

- 용도: 드롭을 바로 지우지 않고 비활성화합니다.

### `/드롭설정`

- 용도: 에너미별 드롭 발생 규칙을 저장합니다.
- 예시:
  - 발생률
  - 슬롯 수
  - 중복 허용 여부
  - 메모

### `/드롭설정보기`

- 용도: 저장된 드롭 설정을 확인합니다.

## 전투 운영 명령

### `/전투보조`

- 용도: 전투 진행 순서를 한 번에 확인합니다.
- 처음 써보는 운영자라면 먼저 이 명령부터 보는 것을 권장합니다.

### `/전투시작`

- 용도: 현재 채널에 전투 세션을 엽니다.
- 예시:  
  `/전투시작 제목:폐공장 추적전`
- 주의:
  - 같은 채널에 이미 열린 전투가 있으면 새로 시작되지 않습니다.

### `/전투종료`

- 용도: 현재 채널의 전투 세션을 닫습니다.
- 특징:
  - 세션 기록과 로그는 남습니다.
  - 세션 안의 열린 참가자 상태는 함께 정리됩니다.

### `/전투참가`

- 용도: 캐릭터를 현재 전투 세션에 참가시킵니다.
- 예시:  
  `/전투참가 캐릭터:레모나`

### `/에너미소환`

- 용도: 에너미를 현재 전투 세션에 소환합니다.
- 예시:  
  `/에너미소환 에너미:슬라임 수량:3`
- 특징:
  - 여러 마리를 소환하면 A / B / C 식으로 표시명이 나뉩니다.

### `/전투상태`

- 용도: 지금 전투판에 올라온 참가자 상태를 확인합니다.
- 진행자 화면:
  - 에너미 체력까지 모두 보입니다.
- 플레이어 화면:
  - 에너미 정보는 최소한만 보입니다.

### `/전투피해`

- 용도: 전투 참가자에게 피해를 적용합니다.
- 예시:  
  `/전투피해 대상:슬라임 A 피해:3`

### `/전투회복`

- 용도: 전투 참가자를 회복합니다.
- 예시:  
  `/전투회복 대상:레모나 회복:2`

### `/전투퇴장`

- 용도: 특정 참가자를 전투판에서 정리합니다.
- 언제 쓰나:
  - 전투에서 빠졌을 때
  - 잘못 올린 참가자를 내릴 때

### `/전투정리`

- 용도: HP 0 이하 에너미를 한 번에 정리합니다.
- 주의:
  - 플레이어는 자동으로 정리하지 않습니다.

## 운영자가 자주 헷갈리는 점

### 1. 왜 `/등록`은 시트 기반이고, 나머지는 DB 기반인가요?

원본 캐릭터 시트는 플레이어가 직접 관리하는 자료이기 때문입니다.  
PANDORA는 원본 시트에서 정보를 읽어오고, 실제 운영 중 조회와 전투와 로그는 DB 기준으로 관리합니다.

### 2. 에너미와 드롭은 왜 비활성화가 있나요?

바로 삭제하면 예전 전투 기록과 운영 이력을 보기 어려워집니다.  
비활성화는 목록에서 숨기고, 필요하면 다시 되살릴 수 있어서 운영이 훨씬 안전합니다.

### 3. 전투가 안 열렸다고 나오면 어떻게 하나요?

먼저 `/전투시작`을 사용하세요.  
그래도 안 되면:
- 같은 채널에 이미 열린 전투가 있는지 `/전투상태` 확인
- 필요하면 `/전투종료` 후 다시 시작
