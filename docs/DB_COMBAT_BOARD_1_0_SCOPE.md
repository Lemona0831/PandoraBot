# PANDORA DB/COMBAT SESSION 1.0 SCOPE

## 문서 목적

이 문서는 `feature/db-combat-1.0` 기준으로 PANDORA 1.0 범위를 다시 고정하기 위한 기준 문서입니다.
README와 충돌하지 않도록, 1.0에서 포함하는 것과 제외하는 것을 명확히 정리합니다.

PANDORA 1.0은 단순 수동 전투 보조만을 뜻하지 않습니다.
1.0에는 다음 두 축이 포함됩니다.

- Supabase / PostgreSQL 기반 데이터 구조 도입
- 전투를 세션 단위로 관리하는 운영 구조 도입

다만 1.0은 턴제 엔진이 아니며, 진행자의 판단을 대체하는 자동 진행 시스템도 아닙니다.
세션, 참가자, HP, 메모, 로그를 관리하는 역극 운영 보조 범위로 한정합니다.

## 1.0에 포함되는 범위

### 1. Supabase / PostgreSQL DB 도입

PANDORA 1.0 범위에는 **Supabase / PostgreSQL DB 도입**이 포함됩니다.

이 범위는 다음을 포함합니다.
- 캐릭터 저장 구조 정비
- 로그 저장 구조 정비
- 에너미 / 드롭 저장 구조 정비
- 전투 세션 / 전투 참가자 / 전투 로그 저장 구조 정비
- Google Sheets에서 PostgreSQL로의 단계적 전환 기반 마련

Google Sheets는 1.0에서 장기 운영용 주 저장소가 아니라,
- 원본 캐릭터 시트
- 초기 확인용 데이터
- 마이그레이션 보조 소스

역할로 축소됩니다.

### 2. 전투 세션화

PANDORA 1.0은 전투를 **세션 단위로 관리**합니다.

즉 1.0 전투 범위는 다음을 포함합니다.
- 전투 세션 생성
- 전투 세션 종료
- 세션별 참가자 추가 / 제거
- 세션별 HP 관리
- 세션별 메모 / 상태 관리
- 세션별 운영 로그 기록

핵심은 “지금 어떤 전투가 열려 있는지”, “누가 참가 중인지”, “누가 얼마의 HP를 가졌는지”, “무슨 조작이 있었는지”를 세션 단위로 정리하는 것입니다.

### 3. 1.0 전투 명령 범위

1.0 전투 범위는 다음 명령을 기준으로 고정합니다.

- `/전투시작`
- `/전투종료`
- `/전투참가`
- `/에너미소환`
- `/전투상태`
- `/전투피해`
- `/전투회복`
- `/전투퇴장`
- `/전투정리`
- `/전투보조`

이 명령들은 세션을 만들고, 참가자를 올리고, 상태를 보고, HP를 조정하고, 전투판을 정리하는 최소 운영 세트를 의미합니다.

### 4. 캐릭터 등록 기준

1.0에서 `/등록`과 `/갱신`은 캐릭터 이름 하나만 믿는 구조가 아닙니다.
원본 Google Sheet를 기준으로 다음 필드를 추적합니다.

- `source_sheet_id`
- `source_sheet_url`
- `source_document_title`
- `imported_character_name`
- `display_name`
- `normalized_display_name`

즉 같은 시트를 다시 등록하면 새 캐릭터가 아니라 기존 캐릭터 갱신으로 처리하고,
시트 내부 이름이 바뀌어도 같은 `source_sheet_id`면 동일 캐릭터로 추적합니다.

## 1.0에서 제외되는 범위

### 1. 턴 / 라운드 / 이니셔티브 / 행동완료

다음 기능은 **1.0 구현 범위에서 제외**합니다.

- 턴 관리
- 라운드 관리
- 이니셔티브 관리
- 행동완료 체크
- 상태이상 지속시간 자동 감소
- 라운드 기반 상태 자동 처리

역극 운영 철학상 1.0에서는 전투의 순서를 기계적으로 통제하지 않습니다.
봇은 세션 운영 도구이고, 진행 순서 판단은 진행자 몫으로 둡니다.

따라서 다음 데이터 구조도 1.0 필수 구현 대상이 아닙니다.
- `current_round`
- `turn_order`
- `initiative`
- `has_acted`

### 2. 성장 자동화

다음은 1.0 범위에 포함하지 않습니다.
- 자동 성장 계산
- 보상 누적 기반 자동 성장 처리
- 장비 / 능력 성장 자동 반영

### 3. 웹-봇 완전 연동

웹 조작 결과가 Discord 알림과 완전히 실시간 연동되거나,
Discord 조작 결과가 웹 UI에 완전 반영되는 수준의 양방향 통합은 1.0 범위에서 제외합니다.

1.0에서는 같은 데이터를 다룰 수 있도록 구조를 정리하는 데 집중하고,
완전한 실시간 동기화는 후속 단계로 미룹니다.

### 4. 커뮤니티 사이트 확장

다음은 1.0 범위에서 제외합니다.
- 플레이어 포털
- 커뮤니티 메인 사이트
- 공개형 포털 기능
- 인벤토리 / 보상 / 기록 조회용 외부 사이트

## 데이터 정책

### 1. 에너미 출현 빈도 삭제

에너미 **출현 빈도** 데이터는 의미 없는 운영 데이터로 판단하고 1.0 범위에서 제거합니다.

따라서 다음 범위에서 모두 제외합니다.
- DB 스키마
- Importer
- 웹 UI
- Discord 명령 출력

### 2. 전투의 운영 철학

1.0 전투는 자동 GM이 아니라 진행자 보조 도구입니다.

봇이 관리하는 것은 다음입니다.
- 어떤 세션이 열려 있는가
- 누가 참가 중인가
- 현재 HP가 얼마인가
- 메모가 무엇인가
- 어떤 운영 조작이 있었는가

즉 세션은 존재하지만, 전투 흐름 자체를 봇이 강제로 통제하지는 않습니다.

## 1.0 한 줄 정의

**PANDORA 1.0은 Supabase / PostgreSQL 도입과 전투 세션 관리를 포함하는 운영 안정화 버전이며, 턴/라운드 없이 세션, 참가자, HP, 메모, 로그를 관리하는 역극 운영용 버전이다.**
