# PANDORA DB / 전투 세션 1.0 범위

이 문서는 `feature/db-combat-1.0` 기준으로  
**1.0에서 어디까지 하기로 했는지** 다시 흔들리지 않게 잡아두는 문서입니다.

중간에 범위가 넓어졌다 줄어들었다 하면 개발도 운영도 같이 흔들리기 쉬워서, 여기서는 기준을 단단하게 적어두는 쪽으로 갑니다.

## 1.0에서 포함하는 것

### 1. DB 도입

1.0은 Google Sheets만으로 끝내지 않습니다.  
운영용 중심 저장소를 PostgreSQL 쪽으로 옮기는 걸 포함합니다.

여기에는 아래가 들어갑니다.

- 캐릭터 저장 구조 정리
- 로그 저장 구조 정리
- 에너미 / 드롭 저장 구조 정리
- 전투 세션 / 참가자 / 전투 로그 저장 구조 정리
- Sheets -> PostgreSQL 이관 준비와 실행

Google Sheets는 완전히 버리는 게 아니라, 이후에는 아래 역할로 줄어듭니다.

- 원본 캐릭터 시트
- 초기 확인용 데이터
- 마이그레이션 보조 소스

### 2. 전투 세션 관리

1.0은 전투를 **세션 단위**로 관리합니다.

여기서 말하는 범위는 이 정도입니다.

- 전투 세션 시작
- 전투 세션 종료
- 참가자 추가 / 제거
- 세션 기준 HP 관리
- 세션 기준 로그 기록
- 세션 상태 확인과 전투판 정리

핵심은 “지금 어떤 전투가 열려 있고, 누가 올라와 있고, HP가 어떻게 변했는지”를 정리하는 것입니다.

### 3. 1.0 전투 명령 범위

1.0 전투 범위는 아래 명령을 기준으로 봅니다.

- `/전투시작`
- `/전투종료`
- `/전투참가`
- `/에너미소환`
- `/전투상태`
- `/전투피해`
- `/전투회복`
- `/전투퇴장`
- `/전투정리`
- `/전투보조`

즉, 전투판을 열고, 참가자를 올리고, HP를 조정하고, 로그를 남기고, 끝나면 정리하는 데 필요한 최소 운영 세트입니다.

### 4. 캐릭터 등록 기준 변경

1.0에서는 캐릭터를 이름만 믿고 저장하지 않습니다.  
원본 Google Sheet 기준으로 추적합니다.

저장 기준은 아래 필드입니다.

- `source_sheet_id`
- `source_sheet_url`
- `source_document_title`
- `imported_character_name`
- `display_name`
- `normalized_display_name`

그래서:

- 같은 시트를 다시 등록하면 새 캐릭터가 아니라 갱신으로 처리
- 시트 안 캐릭터 이름이 바뀌어도 같은 시트면 같은 캐릭터로 처리

가 됩니다.

## 1.0에서 안 하는 것

### 1. 턴 / 라운드 / 이니셔티브 / 행동완료

이건 1.0에서 안 합니다.

- 턴 관리
- 라운드 관리
- 이니셔티브 관리
- 행동완료 체크
- 상태이상 지속시간 자동 감소
- 라운드 기반 자동 처리

이유는 단순합니다.  
PANDORA 1.0은 자동 GM이 아니라, 진행자 보조 도구 쪽에 더 가깝기 때문입니다.

따라서 아래 같은 필드도 1.0 필수 대상이 아닙니다.

- `current_round`
- `turn_order`
- `initiative`
- `has_acted`

### 2. 성장 자동화

이것도 1.0 범위 밖입니다.

- 성장 포인트 자동 계산
- 보상 누적 기반 자동 성장
- 장비 / 능력 성장 자동 반영

### 3. 웹-봇 완전 연동

웹에서 누른 조작이 Discord에 완전 실시간으로 반영되고,  
Discord 조작이 웹 UI에 즉시 1:1 동기화되는 수준은 1.0에서 하지 않습니다.

1.0은 “같은 데이터를 다룰 수 있게 구조를 정리하는 단계”라고 보면 됩니다.

### 4. 커뮤니티 사이트 확장

다음은 후순위입니다.

- 플레이어 포털
- 커뮤니티 메인 사이트
- 공개형 포털 기능
- 인벤토리 / 보상 / 기록 조회용 외부 사이트

## 데이터 정책

### 에너미 출현 빈도는 뺍니다

에너미 `출현 빈도`는 운영상 큰 의미가 없는 데이터라고 보고 1.0 범위에서 제거합니다.

즉, 아래에서 다 뺍니다.

- DB 스키마
- Importer
- 웹 UI
- Discord 명령 출력

## 운영 철학 한 줄

1.0 전투는 자동으로 굴러가는 턴제 시스템이 아닙니다.  
세션, 참가자, HP, 로그를 정리해 주는 **역극 운영 보조 도구**라고 보면 됩니다.

## 마지막 한 줄 요약

**PANDORA 1.0은 Supabase / PostgreSQL 도입과 전투 세션 관리를 포함하는 운영 안정화 버전이고, 턴/라운드 없이 세션, 참가자, HP, 로그를 정리하는 데 초점을 둡니다.**
