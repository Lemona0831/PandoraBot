# PANDORA DB/COMBAT BOARD 1.0 SCOPE

## 문서 목적

이 문서는 PANDORA 1.0에서 데이터 저장 구조와 전투판 보조 기능의 범위를 고정하기 위한 기준 문서다.
1.0은 전투를 자동 진행하는 시스템이 아니라, 진행자가 수동으로 운영하는 전투를 더 빠르고 안정적으로 처리할 수 있도록 돕는 운영 보조 범위를 목표로 한다.

## 1.0에 포함되는 범위

### 1. DB 이전 포함

PANDORA 1.0 범위에는 **DB 이전이 포함된다.**

이는 Google Sheets 기반 구조를 당장 완전히 폐기한다는 뜻이 아니라, 이후 확장과 운영 안정성을 위해 전투판 및 운영 데이터 구조를 DB 중심으로 옮길 수 있는 기반을 1.0 범위 안에서 다룬다는 의미다.

### 2. 전투판 관리 포함

1.0에서는 다음 범위의 전투판 관리 기능을 포함한다.

- 전투판 생성
- 전투판 종료
- 활성 참가자 관리
- HP 관리
- 메모 관리
- 전투 로그 관리

즉, 진행자가 현재 장면에서 누가 전투판에 올라와 있는지 보고, 필요한 대상의 HP와 메모를 조정하고, 운영 로그를 남기는 수준까지를 1.0 범위로 본다.

### 3. 활성 참가자 중심 운영

1.0의 전투판은 서버 전체의 선택 상태를 그대로 보여주는 구조가 아니다.
전투판은 반드시 **현재 활성 전투 참가자만 따로 관리하는 구조**를 사용한다.

따라서 전투 상태 확인 기능은 전체 `/선택` 상태를 조회하는 것이 아니라, 현재 전투판에 올라온 참가자만 조회해야 한다.

### 4. 캐릭터 등록 기준 명시

플레이어의 `/등록`은 단순 캐릭터 이름 기준이 아니라, 다음 정보를 기준으로 처리한다.

- `source_sheet_id`
- `source_document_title`

즉, 1.0 기준 등록 구조는 원본 시트/문서의 출처를 추적 가능한 형태로 유지해야 한다.

## 1.0에서 제외되는 범위

### 1. 턴제 전투 자동화 제외

다음 기능은 **1.0에서 제외**한다.

- 턴 관리
- 라운드 관리
- 행동완료 체크
- 이니셔티브 관리

이 기능들은 전투를 규칙적으로 자동 진행하는 구조에 가깝고, 현재 PANDORA의 운영 방식과도 맞지 않으므로 1.0 범위에 넣지 않는다.

### 2. 성장 자동화 제외

성장 처리, 보상 누적, 자동 레벨업, 자동 성장 계산 같은 기능은 1.0 범위에 포함하지 않는다.

### 3. 웹-봇 완전 연동 제외

웹 대시보드 조작이 Discord 알림과 완전히 동기화되는 구조, 또는 반대로 Discord 조작이 웹 상태를 실시간으로 밀어 넣는 완전 연동 구조는 1.0에서 제외한다.

1.0에서는 웹과 봇이 같은 운영 데이터를 다룰 수 있도록 구조를 정리하되, 완전한 실시간 연동은 이후 단계로 넘긴다.

### 4. 커뮤니티 사이트 제외

플레이어 포털, 커뮤니티 사이트, 공개 웹페이지, 마이페이지, 커뮤니티 기능은 1.0 범위에서 제외한다.

## 데이터/기능 정리 원칙

### 1. 에너미 출현 빈도 삭제

기존 구상에 있던 **에너미 출현 빈도** 항목은 1.0 범위에서 삭제한다.

에너미 데이터는 진행자가 장면에 맞게 직접 선택하고 올리는 구조를 우선하며, 출현 빈도 수치 자체는 1.0 운영 보조에 꼭 필요하지 않다.

### 2. 전투판은 운영 보조 도구로 한정

전투판은 다음을 관리하는 도구로 한정한다.

- 누가 올라와 있는가
- 현재 HP가 얼마인가
- 메모가 무엇인가
- 어떤 조작이 있었는가

즉, 전투판은 진행자의 판단을 대신하는 자동 GM 시스템이 아니라, 수동 운영을 정리하고 실수를 줄이는 보조 도구다.

## 1.0 한 줄 정의

**PANDORA 1.0은 DB 이전을 포함한 운영 안정화 버전이며, 전투 자동화 대신 전투판 생성/종료와 참가자, HP, 메모, 로그를 관리하는 수동 전투 보조 버전이다.**
