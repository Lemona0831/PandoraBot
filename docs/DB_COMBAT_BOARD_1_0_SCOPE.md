# PANDORA DB/COMBAT SESSION 1.0 SCOPE

## 문서 목적

이 문서는 PANDORA 1.0에서 DB 이전과 전투 세션 기능의 범위를 다시 고정하기 위한 기준 문서다.
1.0은 단순 수동 전투 보조를 넘어서, **Supabase/PostgreSQL 기반 저장 구조 도입**과 **전투를 세션 단위로 관리하는 운영판**을 포함한다.

다만 1.0은 전투를 완전 자동화하는 시스템이 아니며, 진행자의 판단을 대체하지 않는다.  
진행자가 세션을 열고, 참가자를 올리고, HP와 메모와 로그를 관리하는 구조까지를 1.0 범위로 본다.

## 1.0에 포함되는 범위

### 1. DB 이전 포함

PANDORA 1.0 범위에는 **Supabase/PostgreSQL DB 도입이 포함된다.**

이는 운영 데이터와 전투판 데이터를 Google Sheets만으로 계속 유지하는 대신,  
이후 확장 가능한 구조로 옮기기 위한 실질적인 이전 작업을 1.0 범위 안에서 다룬다는 뜻이다.

### 2. 전투를 세션 단위로 관리

PANDORA 1.0은 전투를 세션 단위로 관리한다.

즉, 1.0 전투 범위는 단순히 활성 참가자 몇 명을 임시로 적어두는 수준이 아니라 다음을 포함한다.

- 전투 세션 생성
- 전투 세션 종료
- 세션별 참가자 관리
- 세션별 HP 관리
- 세션별 메모 관리
- 세션별 운영 로그 관리

### 3. 1.0 전투 명령 범위 고정

1.0 전투 범위는 다음 명령을 기준으로 고정한다.

- `/전투시작`
- `/전투종료`
- `/전투참가`
- `/에너미소환`
- `/전투상태`
- `/전투피해`
- `/전투회복`
- `/전투퇴장`
- `/전투정리`
- `/전투보조`

이 명령들은 “전투 세션을 열고 닫고, 누가 올라와 있는지 확인하고, HP와 메모와 정리 상태를 관리하는 운영판”을 이루는 최소 범위다.

### 4. 캐릭터 등록 기준 명시

플레이어의 `/등록`은 캐릭터명 문자열 하나만 저장하는 구조가 아니다.
1.0에서는 다음 정보를 기준으로 저장 구조를 유지한다.

- `source_sheet_id`
- `source_document_title`
- `imported_character_name`
- `display_name`

즉, 원본 Google Sheet의 출처와 가져온 당시 캐릭터명, 표시용 이름을 구분해서 저장해야 한다.

## 1.0에서 제외되는 범위

### 1. 턴/라운드/행동완료/이니셔티브 제외

다음 기능은 **1.0 구현 범위에서 제외**한다.

- 턴 관리
- 라운드 관리
- 행동완료 체크
- 이니셔티브 관리

따라서 다음과 같은 데이터 구조도 1.0 필수 구현 대상으로 두지 않는다.

- `current_round`
- `turn_order`
- `has_acted`
- initiative 관련 필드

즉, 전투는 세션 단위로 관리하지만, 진행 순서를 기계적으로 추적하는 턴제 시스템으로 만들지는 않는다.

### 2. 성장 자동화 제외

성장 처리, 보상 누적, 자동 레벨업, 자동 성장 계산 같은 기능은 1.0 범위에 포함하지 않는다.

### 3. 웹-봇 완전 연동 제외

웹사이트 조작 결과가 Discord 봇 알림으로 완전히 실시간 반영되거나, 반대로 Discord 조작이 웹 상태를 즉시 밀어 넣는 완전 연동 구조는 1.0에서 구현하지 않는다.

1.0에서는 웹과 봇이 같은 데이터를 다룰 수 있도록 구조를 정리할 수는 있지만, 완전한 알림 연동은 이후 단계로 넘긴다.

### 4. 커뮤니티 사이트 제외

플레이어 포털, 커뮤니티 사이트, 공개 웹페이지, 마이페이지, 커뮤니티 기능은 1.0 범위에서 제외한다.

## 데이터 정리 원칙

### 1. 에너미 출현 빈도 삭제

에너미 **출현 빈도** 데이터는 의미 없는 데이터로 판단하고 1.0 범위에서 제거한다.

따라서 다음 범위에서 모두 제외한다.

- DB 스키마
- Importer
- 웹 UI
- Discord 명령 출력

에너미 데이터는 진행자가 장면과 상황에 따라 직접 선택하고 소환하는 구조를 우선한다.

### 2. 전투판은 진행자 운영판

1.0 전투판은 자동 GM이 아니라 진행자 운영판이다.
전투판은 다음을 관리하는 도구로 한정한다.

- 어떤 세션이 열려 있는가
- 누가 참가 중인가
- 현재 HP가 얼마인가
- 메모가 무엇인가
- 어떤 운영 조작이 있었는가

즉, 세션은 존재하지만 전투 흐름 자체를 봇이 강제로 통제하지는 않는다.

## 1.0 한 줄 정의

**PANDORA 1.0은 Supabase/PostgreSQL DB 도입과 전투 세션 관리까지 포함하는 운영 안정화 버전이며, 턴/라운드 없이 세션, 참가자, HP, 메모, 로그를 관리하는 전투 운영판 버전이다.**
